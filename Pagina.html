Conceptos Clave del API Gateway

Un API Gateway actúa como un intermediario o "puerta de entrada" para todos los servicios internos de una aplicación. No solo dirige el tráfico, sino que realiza un conjunto de tareas cruciales. Haz clic en cada tarjeta para conocer sus funciones principales.

Comparativa: Forward Proxy vs. Reverse Proxy

Para entender un API Gateway, primero hay que diferenciar los dos tipos principales de proxies. Aunque ambos son intermediarios, su función y perspectiva son opuestas. Usa el interruptor para explorar cada uno de estos **conceptos fundamentales**.

Forward Proxy

Cliente 💻 --petición--> Forward Proxy 🛡️ --petición--> 🌐 Servidor Externo

Ejemplos de uso: control de acceso en empresas, anonimato personal.

Reverse Proxy

Cliente 💻 --petición--> Reverse Proxy 🚪 --petición--> ⚙️ Servicio Interno

Ejemplos de uso: balanceo de carga para sitios web, seguridad de servidores.

Conclusión Clave: Un API Gateway es una forma avanzada de **Reverse Proxy**, ya que actúa en nombre de los servidores. Sin embargo, añade mucha más inteligencia a nivel de aplicación, como veremos en la siguiente sección.
